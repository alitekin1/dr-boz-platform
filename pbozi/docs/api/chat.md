# Chat & Agent API

The core interaction layer for AI messaging and tool execution.

## Standard Chat

### 1. List Chats
- **URL:** `GET /chats`
- **Query Params:** `project_id`, `user_id` or `telegram_user_id`

### 2. Send Message (Non-Streaming)
Processes the message, executes tools if necessary, and returns the final assistant reply.
- **URL:** `POST /chats/{chat_id}/messages`
- **Body:**
  ```json
  {
    "content": "Hello, can you analyze my project?",
    "model_id": 1,
    "telegram_user_id": 123456
  }
  ```
- **Response:** Includes `message` (content, reasoning), `tool_calls`, and `context_used` (RAG status).

### 3. Download Generated PDF
Retrieves a PDF generated by AI tools.
- **URL:** `GET /generated-pdfs/{filename}`

## Agent Chat (Advanced / Streaming)

The Agent API uses **Server-Sent Events (SSE)** for real-time streaming of content and tool execution logs.

### 1. Start Agent Chat
- **URL:** `POST /agent/chat`
- **Body:**
  ```json
  {
    "message": "Analyze the sales data and create a chart",
    "chat_id": 5,
    "thread_id": "unique_session_id"
  }
  ```
- **SSE Events:**
  - `type: content`: Incremental text chunks.
  - `type: tool_start`: Notification that a tool is being called.
  - `type: tool_end`: Result of the tool execution.
  - `type: usage`: Token usage metadata.
  - `type: done`: Stream completion.
